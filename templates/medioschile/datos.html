Los datos recopilados en Mapa de Medios están disponibles para la comunidad (ciudadanos, estudiantes, investigadores, etc.) bajo una estructura de Open Data. Esto implica que los datasets se encuentran bajo una licencia abierta y se tomaron decisiones técnicas sobre los formatos e instauramos una política de metadata adecuada. Los tres aspectos están unidos entre sí y son necesarios para que los datos sean catalogados como Open Data y, tan importante como eso, puedan ser usados por la comunidad.

Según la definición de Open Data de Open Knowledge Foundation los datos "deben poder ser leídos automáticamente, estar disponibles de una sola vez y proporcionarse en un formato abierto (es decir, un formato que tenga sus especificaciones disponibles pública, libre y gratuitamente, que no imponga restricciones, económicas o de otro tipo sobre su uso) o, al menos, que puedan ser procesados con una herramienta de software libre" ( http://opendefinition.org).

En lo que respecta a los formatos elegidos para hacer disponibles los datos, tomamos la definición de freedomdefined.org: "Para los ficheros digitales, el formato en que se haga disponible el trabajo no debe estar protegido por patentes, salvo que se conceda un permiso libre de regalías, ilimitado, irrevocable y de ámbito mundial para hacer uso de la tecnología patentada. Si bien en ocasiones se pueden usar formatos no-libres por razones prácticas, debe haber disponible una copia en un formato libre para que el trabajo sea considerado libre."

Ambas definiciones son ampliamente usadas en todo el mundo como condiciones para el open data.

Licencia

Los datasets generados en esta investigación están protegidos por la licencia Open Database License (ODbL) v1.0.

Esta licencia permite a las personas:

  • copiar, distribuir y utilizar la base de datos
  • producir obras a partir de la base de datos
  • modificar, transformar y construir sobre la base de datos

Asumiendo los siguientes compromisos:

  • dar crédito a la fuente para cualquier uso público de la base de datos, o de las obras producidas a partir de la base de datos
  • dejar siempre en claro, para cualquier uso o redistribución de la base de datos o de los trabajos producidos a partir de ella, la licencia que la protege
  • todas las versiones adaptadas o trabajos derivados de la base de datos se deben ofrecer bajo la misma licencia
  • si se distribuye la base de datos o una versión adaptada de ella bajo medidas tecnológicas que restringen el trabajo (como DRM) también se debe distribuir una versión sin esas medidas tecnológicas

Formato

Los datasets están publicados en primera instancia en Open Society, Poderomedia: Poderopedia Chile y Colombia, Consejo de Redacción, y OpenData Latinoamérica. Gran parte de la información recabada en la investigación fue guardada en spreadsheets, por lo que su publicación en descargables puede ser en excel, csv, json y xml.

En la elección de los formatos se optó por formatos abiertos y no privativos. Esto significa que los datos pueden ser descargados para ser revisados y procesados por lo menos con un programa de software libre. Esto se hace extensivo a toda la entrega de la investigación. En la siguiente lista se indican los formatos usados:

Formatos abiertos:

  • Texto → ODT (OpenDocument Texto)
  • Imágenes → PNG (Portable Network Graphics)
  • Páginas WEB → HTML (HyperText Markup Language), CSS, DOM
  • Audio → Ogg
  • Multimedia → WebM
  • Multimedia → Mkv (Matroska Multimedia Container)
  • Datos → CSV (comma-separated values)
  • Datos → JSON (JavaScript Object Notation)
  • Datos → XML (Extensible Markup Language)

Otros formatos, que pueden ser abiertos por programas libres:

  • Texto → DOC (Microsoft Word)
  • Datos → XLS (Microsoft Excel)
  • Documentos → PDF (Portable Document Format)

La elección de los formatos depende de la naturaleza de la información que se recopiló durante la investigación, por lo que no todos los formatos de la lista fueron utilizados.

Metadata

Cada página publicada en Internet que incluya un dataset fue etiquetada con la información correspondiente que permite a las personas y programas informáticos (por ej. buscadores):

  • identifica esa página como contenedora de un dataset
  • obtiene información de los datos disponibles tales como: identificador único, autor, mantenedor, lenguaje, fecha, período cubierto por los datos, lugar geográfico, actualización. Estos datos, entre otros, son los que transforman un dataset en linked data.

Para publicar datasets en páginas web son necesarias las siguientes etiquetas básicas, que varían según el dataset.

Etiquetas básicas a usar en los dataset del proyecto

  • Nombre del dataset → property='dct:title'
  • Descripción del dataset → property='dct:description'
  • Lenguaje del dataset → property='dct:language'
  • Fecha de publicación del dataset → property='dct:created'
  • Organización/Persona generadora del dataset → property='dct:publisher'
  • Cobertura temporal del dataset → property='dct:temporal'
  • Cobertura geográfica del dataset → property='dct:spatial'
  • Formato del dataset → property='dcat:mediaType'
  • Tamaño del dataset → property='dcat:byteSize'
  • URL del recurso → property='dcat:accessURL'
  • Frecuencia de actualización del dataset → property='dcat:accrualPeriodicity'
  • Licencia del dataset → property='dct:license'
  • Palabras claves para ubicar el dataset → property='dcat:keyword'

El etiquetado adoptado es el de Dublin Core Metadata Initiative y propuesto por el Open Data Institute con sede en Londres, UK: http://dublincore.org/documents/dcmi-terms/
